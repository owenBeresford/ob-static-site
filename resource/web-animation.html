There are many, many pages on the interwebs explaining 'how to animate'. In this text, “to animate” is “to make a thing move”, maybe colour, maybe location, maybe size, etc. I already know how to animate in raw JS or raw CSS ~ for most of the other articles I gain information.

Animation policy (after general reading, particularly W3C)

  • As described in favicon-notes, animation MUST be used wisely and to say something, NOT spamming effects.
  • Animation MAY be used to draw attention to elements.
    • Animation MAY be used to show content changes more clearly ~ this actually helps.
  • Sometimes, animation is used to make pages more cute (not applicable to my work).
  • Animation MUST NOT block UI or user desires. Animation MUST NOT slow the load stage.
  • Animation MUST NOT make interactions harder to use, for example moving buttons away from the user's mouse,
  • Animation sometimes triggers medical problems, so there MUST be a disable option.

Business need: Animation options

As a high-level checklist, the following options exist:

  • actual video feeds / small static video assets
  • use SVG animation features see svg-animation
  • use JS to imperatively edit HTML elements
    • OR SVG elements
  • use CSS animation to declaratively edit HTML elements
    • OR SVG elements
  • HTML canvas + JS (different angle) 1
  • webGL 2 (real honest 3d + flat batteries on phones) 3
  • use CSS transitions to declaratively edit HTML elements
    • OR SVG elements
  • scroll animation, so idle when users are reading (CSS only, AFAIK).
  • ...
  • using JS to swap images (whole images, not sprite animation). These will need to be cached locally to keep performance.
  • sprite animation (this will never die, but is slow to build, unless auto-code) OLDER
  • GIF animation 4 / MNG animation 5 OLDER, INFERIOR
  • HTML refresh 6 and different HTML (alot OLDER, INFERIOR)

Animation can be:

  • routine, applying standard effects, and looking like Powerpoint. In this case using a standardised framework is better use of time.
  • for specific interactions, a locally written one delivers the results better. As an example the small animation on my homepage is not regular, and not repeating frequently.

CSS keywords 7, list of HTML properties that can be animated 8

  • As background, MSword (and therefore Powerpoint or Visio) doesn't natively do motion in curves, or curve shapes well.
  • SVG is much better for curves than HTML is. HTML Borders can be curved in HTML, but there are annoying edge cases that fail.
  • Animation via JS or CSS can be from a framework based or locally written. The “locally written” version, can be generated by hand, with pre-LLM tools (see list below), or an LLM.

QA and UXA on animation.

  • I can't see test tools specifically for animation. I or other devs can build the outcome on top of Lighthouse as a in-browser test docs 19.
  • Need a magic trick 20 to disable on slower CPU/ overly busy CPU/ low battery machine. An article CLAIMS the browser will say when its low battery in media-queries 21 ~ I haven't seen this happening.
    • In JS, there is battery status API for many users 22, but not supported by Safari 23. There is a fairly old component for this 24 or in plain React 25. This is a more frequently updated module 26. One of these JS libraries can be used to set a CSS class for “UI effect reduction”, including animation.
  • Obviously pay attention to user requests eg prefers-reduced-motion 27, accessibility-testing.
  • Matching 'banner blindness' 28 by routine web users, permanently scrolling bylines are likely to be ignored.
    • Positive uses for scrollers is to allow titles to big to fit on screen to be accessible, and 1 line. This situation is most common on mobile.
  • Animation for good for UX (eg instant DOM changes can baffle slower users) 29 30
  • Obviously browser compat testing, and phone testing need to happen regardless
    • For example, a successful app-dev turned rel-dev known as PPK 31, has a test suite of ~50 devices, so delivers high quality apps.
    • Note animating colours is awkward on Safari, any float alpha values are converted to black. See colourise-emoji, search for Safari in 32.
