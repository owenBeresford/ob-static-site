Simple and boring list, pweeze don't fall asleep whilst reading it.

I define DONE as :

  • enough unit tests;
    • unit tests on all the complex code
    • an availability (url verify) unit
    • security access unit
    • a list of user journeys, and a BDD type tests that verifies it
    • “page load speed”, for an end-users definition of “page”
    • a stress test, to be run carefully, maybe second DB
    • ACTION: standard unit test library can do this 1 2 3; add libraries to support the relevant domain problems
  • run result HTML through html linter
    • add labels elements
    • add titles attribute (I don't think that mobile exclusive websites make sense, there are many apps that cannot be installed to large screen devices, but that is not a website)
    • add languages attribute
    • alt attribute
    • add meaningful noscripts, maybe to suggest another user-journey option
    • using HTML5 properly, try html5-tags
    • ACTION: can use tidy 4 on CLI, can use 5 if HTML is publicly accessible. Items in npmjs seem to use one of these two choices.
    • ACTION: on me to build a list of tag nesting requirements, like a eslint config
  • check UI is accessible
    • contrasting colours
    • “buttons” (whatever technology) are large enough for thumbs
    • actions are labeled, and current state is visible (people can be distracted and using a PC/mobile)
    • user goals are clearly signposted
    • interaction simplicity (when not a game)
    • forms must work on phones, in that when the phone soft-keyboard is visible, the form is still usable
    • forms shouldn't perfectly reproduce a previous paper-tech process; where an interaction can be automated, it should be (e.g. postcodes first with a PAF for the rest of the address)
    • Vet modern HTML compliance, i.e. DIALOG, POPUP tags
    • text inputs should support pasting correctly
    • NOBODY SHOULD EVER FORCE A USER TO TYPE A POSTCODE IN CAPITALS MANUALLY. Yes I'm shouting. This is not a typewriter.
    • I do not have a reliable technology to describe the above bullets in this section. Cucumber/ Gherkin / Behat /Selenium are a start (and/or JS tooling in BDD mode). Try storybook.
  • cyclomatic complexity ?
    • 6 jetbrains tooling 7
    • Article not intensionally an advert for Jetbrains tools. Other vendors are available.
  • Spelling for human text in the source-code
    • 8 9 Jetbrains tooling
    • Note: It is sometimes useful to pull-out the text into a config to make rebranding faster
