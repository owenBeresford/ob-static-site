This is a highcharts demo 1 2, as this seems a popular graph library. Highcharts is another library that builds SVG graphs, so they are fairly easy to style and encourage to be responsive. I made a test page similar to this at Hoptroff for several graphing libraries, but that is company source. The first render of this graph only took a short while; I am focussing on branding things not on the data; which is why I used data from the library documentation.
As I have quickly built this demo, and I don't think this code was hard or impressive. I don't perceive risk in starting a role that uses this graph library rather than the one I used at Hoptroff.
As a second feature on the demo; as far as I can see all the animation effects are just applying the data that you already have onto the screen 3 4 5. Therefore in order to do useful updates the traditional solution using setIntervail(), or another event stream will be needed. This now has colour blend support, Highcharts integrates the LinearGradient tag 6. The visible text on the graph isn't copied and pasted from a German language website; I typed the new labels directly, and ran a spell check.

  • Highcharts is mostly just a graphing library in JS, so will score low on my standardised feature comparison list (see data-visualisation-epic ). There are many articles using python as a data-source for highcharts; I cannot see anything that lets you build highcharts graphs from the python interpreter. Highcharts is fairly mature and currently supports 7:
    • arcdiagram, area, arearange, areaspline, areasplinerange, bar, bellcurve, boxplot, bubble, bullet, column, columnpyramid, columnrange, cylinder, dependencywheel, dumbbell, errorbar, funnel, funnel3d, gauge, heatmap, histogram, item, line, lollipop, networkgraph, organization, packedbubble, pareto, pie, polygon, pyramid, pyramid3d, sankey, scatter, scatter3d, series, solidgauge, spline, streamgraph, sunburst, tilemap, timeline, treemap, variablepie, variwide, vector, venn, waterfall, windbarb, wordcloud, xrang
    • use of graphing libraries will lead to knowledge on what the abbreviations mean; for other readers, these terms are defined in the preceding link.
  • Highcharts has a middleware (like grafana) 8 , but its expected data source is Excel. Look at the diagram on here 9 if unclear about architecture. UPDATE: in 2020 they renamed the API to Everviz. I have searched online, I have not seen a reason as to why use the API; I guess if your output is presented on a small phone, creating the graphs on static/ fixed power supply is better experience. It would be cheaper to use Node on your own server; then render and save the graphs as static assets. All my work involving graphs has been dynamic data, or least fairly-short shelf-life data; so managing caching on the static assets would be important.
  • Highcharts supports time-series graphs, as demonstrated in 10. It also has a long list of other rendering output formats (left menu on same link) example, non-GIS heatmap 11 honeycomb plot 12. The zoom feature is easier to use than other graphing libraries; unless you want data at different granularities per zoom (first link shows this).
  • As a graphing library, Highcharts only supports graphs; but can be used with any DB if you create some middleware. Highcharts has plugin support, and a list of user created plugins 13.
  • As is typical, Highcharts supports style overloads via extra CSS 14, When used with other software (e.g. the middleware, or Odoo 15 ) Highcharts can create dashboards; the tools blog has a PNG of a dashboard but no blurb, so I don't think this a feature that they emphasise.
  • Highcharts doesn't export alerts; but you could access the render cycle, and create your own 16 or do this in the data sourcing API, which is better data architecture. Highcharts doesn't have monitoring features, but can be used for monitoring. Various people have created utilities, however I haven't seen any specific maths libraries for Highcharts; example 17 18. Highcharts has mapping output format 19, but isn't a GIS. Likewise it has no NLP or data projection (i.e. future data)
