I am getting annoyed when I discover another HTML5 tag. This should be a sample of each tag, for copying. I have removed the HTML[234] items from this list.
  • <address> ~ [not actually new, but I don't use it much] tag for an address, normally ownership of current document
  • <article> ~ marker for actual content, not HTML fluff
  • <aside> ~ HTML5 but erm? not article content, but related
  • <audio> ~ ....
  • <button> ~ obvious. THIS SHOULD ALLOW A HREF.
  • <canvas> ~ As every other windowing platform
  • <caption> ~ Not HTML5, is a label that can be applied to tables.
  • <cite> ~ Title/ reference/ biblio
  • <data> ~ A way to embed data for the computer, to make scripting easier
  • <datalist> ~ A tag like select, but to allow text input to have autocomplete, for predefined values.
  • <details> ~ Pairs with summary, this tag is the long version
  • <dfn> ~ A tag to describe that you are defining a term.
  • <dialog> ~ Currently poor browser support. Better browser version of forms, from the look of it. UPDATE 2023: browser support has increased
  • <embed> ~ parallel tag to video or audio, is HTML5
  • <figcaption> ~ To match a figure, is HTML5
  • <figure> ....
  • <footer> ....
  • <keygen> ~ Not sure about the use of this...
  • <main> ~ same idea as article
  • <mark> ~ that annoying thing they put in blogs "top hi-lite"
  • <menu> ~ HTML5 menu, to reduce UL + CSS
  • <menuitem> ~ see previous
  • <meter> ~ a slider for ranges, not singular items
  • <nav> ~ a tag to say what is in the navigation
  • <object> ~ not HTML5, used to link multimedia to a webbpage.
  • <output> ~ to display results
  • <param> ~ item for object tags
  • <progress> ~ a meaningless tag for progress bars (I think file-save processes, or file-download processes are overused and do not communicate anything. Useful for progress through a form)
  • <samp> ~ not HTML5, and I never use this
  • <source> ~ a sub tag for audio/ video/ picture tags
  • <summary> ~ combines with details, modern menu-ing
  • <template> ~ as it says, not sure of value. docs say content is not in the DOM
  • <time> ~ for representing the time
  • <track> ~ thing for meta data, in time tracking objects
  • <var> ~ ...
  • <video> ~ ...
  • <wbr> ~ line break opportunity.
  • <search> ~ currently the next Safari only, but semantically tell browser this HTML island is search feature
  • clickable geolocations ~ Some browsers support 'geo:' as an URL type, to match 'emailto:'. A link like this should do a click through to your devices mapping software. example; <a href='geo:51.26769008051434, 0.079430025246794'>Our location</a>
  • clickable phone numbers ~ some business's like to provide a phone number, this can be interactive from their brochure site.
