NB: If you are an US citizen you will be confused by the 'n' and 's' in 'Columniser'; at the point I work in the US branch of a US company I will change the spelling.
CSS3 support allows a different approach, this is not new. The raw CSS ‚Äúcolumns‚Äù aggregate declaration is supported by 96% 3, and ‚Äúcolumn-count‚Äù is at 97.2% 4. I have made the elipsis as a CSS feature a number of years ago so that has no need to change. As I have many numbered lists, I would like keep the numbering working (see below tests). I have a CSS sample which should manually manage the counts in any case.
I haven't looked at grid for article, as that will not apply columns to a large single paragraph of text.
