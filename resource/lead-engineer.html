This role is atypical for a contract, and is hidden behind an NDA. I performed for Venue10, as a two line summary:

  • As team-lead, co-ordinated geo-diverse development team. Acted as product vision in places.
  • Created/ added standard technology to improve services, towards standard outcomes. This is not a tech-centric company. I delivered alot of targets.

Activities for UI development

  • Co-ordinating an international tech team, with people in three countries.
  • Managed/ lead extra remote-contractors, in several further countries. This required managing best communication times for separate working hours.
  • Increased stakeholder awareness and requested advise on product user-flow.
  • Establishing standard agency work flow, with CI and concurrent git branches, moving towards weekly sprints. The sprints are to manage task coordination between developers; weekly releases are needed as small team.
  • Added user-stories to improve communication and values. Built culture of ROWE development, as I was hired after expected release date.
  • Emphasised DDD to improve communication with non-technical directors. Liaised directly with directors, as this is a small startup.
  • I rebuilt the actual servers and architecture, simultaneously reducing costs and increasing performance and stability. Managed these cloud boxes.
  • I managed compilation/ deployment of iphone apps.
  • We added all the SEO/ web semantics features, which for pre-commercial milestones where omitted. Added marketing related data-features.
  • I created and managed CORs headers. Created some pure frontend postMessage() based communication, for other features.
  • Created a new REST API (replacing earlier hard-to-read RPC) for better programmatic interaction.
  • Added testing on the real time messaging feature in the booking engine, so it was shown to work with up-to 10,000 daily users. This is a small startup, and this number was a growth target for my entire employment. The booking engine is one of the external contracts, and expected to be a critical bottleneck for scaling. It is expected that 90-95% of traffic will be in a 1-2h window per day.
  • Improved OO and DB interaction for better stability. My new code followed more recent PHP language features.
  • For the code that I wrote, I followed standard engineering practice ~ TDD ~ as this enabled complex requirements to have timescales.
  • The platform visibly had strong search features. I wrote the second generation on these, replacing all previous editions. Particularly I improved performance (about 1000%), as this had been left till the end. Secondly focussed on usefulness and user stories, so the search was relevant to consumers. GIS was also critical part of the search, this is purely my work.
  • Although I did improve the search, I also moved the company's perception of search from a database resultset to a shortlist of viable options. I moved the search filters to ones more useful to users, and tried to avoid attention on price sorting (as it was nonsense stats). A particular meal and drinks has a defined price, but this isn't the “average price” across the venue, unless you intend to order the same meal from every venue (regardless of cuisine).
  • In Jan 2016, I was trying to get time to implement the third generation of search, which would be faster and more of the above ideas.
  • Analysed and implemented a long list of UX improvements, to reduce data entry requirements for users. This is an application, not a set of webforms. In total this reduced human inputs by about 20% without any loss of features. Changed UX widgets to ones that require less clicks. For example there should be no need to set a [long,lat] manually, when the platform has the physical address.
  • Did alot of “idiot testing” and found missing feedback or interactions, to make things simple from the users perspective. I entirely defined task definition to being defined as user journeys, to reduce rate of missing interactions.
  • As milestones have been achieved; I reversed the previous ROWE work style to make code easier to manage.
  • Due to extending requirements I created a multi-level user authentication system.
  • Did about several months of “no new features” quality refinement. This lead to better UX and improved client side performance.
  • Wrote some business features in Node. The most complex code that I have written at that date in Node, and computed available booking slots, and characteristics of them. Built way to generate tests quickly.
  • Performed code audits to improve non-functional things like API authentication; these were my most important changes done on the code that I hadn't written.
  • Was planning on moving to Symfony for better features and performance, but this did not happen due to budget limitations/constraints.
  • Worked on bid on TechStar accelerator funding, in the “food / hospitality tech” category. This was in the incubation period, the competition, Some Notes.
  • I worked part-time for venue10 for the final two months, as most of the work was done (for example I got paid to write/ publish API documentation). Once in my next role, I was retained for a small number of hours to perform maintenance operations.

For the first three months, my job title was “lead developer”, afterwards I was “lead engineer”. This role had the least software engineering work of any that I have held, due to other responsibilities. To further the confusion, the company rebranded to dinestream.com in late 2016.
