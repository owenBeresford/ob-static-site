My missing knowledge gap

  • 'g' tag is for group
  • Just like XML, things go horribly wrong if you have mismatched tags.
  • Add 'DOCTYPE' header by hand if needed.
  • You can inline SVG elements, attach to the document with an IMG element, or an OBJECT element.
  • In 2017, if you needed to script the second two ways to embed the image, you needed an extra documentElement lookup as it was a separate document object. In 2023, in particular using querySelector, you can directly access the elements inside the SVG as a single step.
  • when using viewBox, the numbers are size of the canvas that you have (in the SVG)

Manual optimisation steps

Activities to reduce file size, so SVG images load faster, and to make the files easier to rebrand in future. There are similar articles 1 2 3 4. This article doesn't mention complex colour blends/ gradients, which can slow down apps on phones; mostly as I haven't needed to use blends that much.

  • If you are using a SVG generated by converting from a different format, only use SVG converters that are smart and applying BÃ©zier curves. There are some clip-art sites that use a poor re-renderer and the files are mahooosive and awkward.
  • Read the XML yourself, rather than spitting it at the webrowser.
  • Remove items that you don't seem to need, or are tagged as display=none.
  • Remove accidental <cntl- D> elements, which are identical to the next node in the file.
  • Pull out style items into defs.style element, especially if the tools have duplicated style attributes 100s of times.
  • Create defs.$name objects and <use $name> to vastly reduce duplication of tags.
  • CSS CSS cheat sheet and style spec for SVG.
  • When SVG is made by image editing tools, strip random tags/ attributes, carefully.
  • Trim unnecessary numerical precision. The numbers are in pixels, so can be integers.
  • Multiple transforms can be applied in the same group tag, but also in the same attrib. The matrix transforms the best for this, but are harder for humans to edit. As with many graphics pipeline tools, the order of the transforms is very important (test this to see).
  • If hand editing, edit ID attributes so that they make sense to a human. This isn't really an optimisation ~ except for your dev-time ~ when scripting events or animations.
