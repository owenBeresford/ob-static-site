This is a part of a range of ‚Äúvisual samples / maquettes‚Äù. Since I started doing more front-end work, I have been making these to speed up browser testing. #boring My thought, I should add a snoring backtrack to these pages.
Mobile apps are the biggest UI and sales opportunity. Mobile apps have small screens, and strongly benefit from extra process to convey what the app is supposed to be doing, or asking for. Mobile apps traditionally do not have tooltip displays, but what if they did in a special fashion. A one word input label allows more space, and a longer contextual message ~ most loudly for password requirements ~ should improve usability. This is a maquette, see how it flows.
This maquette should explore keyboard pop-ups and avoidance. As a mobile-first design, there are few inputs adjacent to each other.

NOTICE: Nothing is uploaded to any servers, it's a UI demo.

This maquette only makes sense on a mobile.
