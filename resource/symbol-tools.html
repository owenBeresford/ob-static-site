Objectives

I have been using font-awesome for a number of years. Since everything changes, should I change this item of tech? 5 goals to take into account for this decision:

  • A way to have icons in a web app; which do not have high management overheads.
  • A way not to have a big list of HTTP requests.
  • A way to have all icons in the same visual/drawing style (i.e. I want the whole palette by same author).
  • A way for things not to look jaggedy (i.e. Vector formats).
  • if using sprite blitzing, must be simple to drive (no hand defined pixel offsets).

7 Contenders:

  • Font awesome 1 (AFAIK, the first one). This has ~4,000 glyphs as a free product, and ~8,000 as paid. This is a font encoded in several formats, and a CSS style sheet to apply singular UTF8 glyphs via CSS :after :content. This is a corporate product, with a freemium version; not open source. Claims to have 38% of market for 3rd party fonts 2
    • good as: fairly easy to use.
    • bad as: quite a large download (particularly when using 2-3 symbols). I have never worked at a company with a large enough tech budget to get the logo in the project.
  • Iconspedia 3 is a big catalogue of icons. Apply images to your brochure/app with an IMG element.
    • good as: much easier search tools, most items are free to use.
    • bad as: they are PNG/ICO, not vector, many of these are separate images, not a structure .
  • Fontisto git , totals ~600 icons, is OSS, MIT licence. Historic site is offline, suspect frozen project
    • good as: is open source, same idea as FA, but is a 6.62 MB download.
    • bad as: only 600 odd icons. According to NPM, in 2023 initially ~600 icons.
  • Fontello 4 git. A tool to select glyphs from other open source fonts, and make a new individual font.
    • good as: will lead to small downloads. Allows joining several other libraries, or your own SVG drawing.
    • bad as: ? um? Nothing so far. Need to iterate the SVG files abit, see end.
  • Icons-8 5 git. This is open-source, but a payfor service.
    • good as: These are in colour. The iOS set is OK design.
    • bad as: Pay for. OSS version is only 312 icons; last update was three years ago (i.e. 2017).
  • MaterialUI needs to be mentioned, but isn't really a matching solution. MaterialUI supplies Components for SPA apps, rather than fancy font tech. I omitted this from the first version of this file as I was just looking at fonts for use outside of SPA.
    • good as: nice for SPA
    • bad as: poor when not making an SPA
  • UPDATE2023 Phosphor icons 6 is a younger project, by someone who wished to avoid the increasing commercialisation of Fontawesome. MIT license. 'Assets' are currently in git. The SVG icons can obviously be imported into JS/TS files in an SPA 7, then fixed with Webpack (or similar build-step).
    • Good as: has line, heavy line and two-tone (i.e. shaded) variants. It has >7000 icons as vector images. Impressively has clean integration to modern frameworks (React, Vue etc) in a fashion that feels like MaterialUI, see docs. Written in Typecript.
    • Bad as: if not using Typescript, this is Typescript. Icons will need to be merged into Fontello individually for use with my current process. But this is what I am doing with the hand-drawn/ edited ones.
  • frIconix 8 Got seriously put off by Quantcast cookie banner, stopped looking.

Chosen Result: Fontello

I have successfully applied Fontello to this website. Fontello is fairly easy to use, but the search tool isn't the obvious to first time users. You can't use ‚Äúfind in page‚Äù, which is how I drive the FA cheat-sheet. It easily makes a zip of your selected glyphs, in a full range of font encodings. The zip also includes CSS to use the fonts (similar to font-awesome.css). The CSS has a ‚Äúfont-" prefix, rather than the FA prefix of ‚Äúfa-"; as an architecture this is good, so you can use both without clash; but as a drop in replacement, this is bad.
PLEASE NOTE: the CSS changes all icons ending in ‚Äúsquare‚Äù to ‚Äúsquared‚Äù. I edited it back to ‚Äúsquare‚Äù in my copy.
In the same change, I swapped the burger menu glyph for a better one [IMO], which I edited first. The process of applying a glyph to Fontello is awkward as it will only accept a narrow subset of SVG for a new glyph. I can't find where the constraints are documented, so the process is ‚Äúiterate until no Errors‚Äù. Drawing an SVG in a diagram tool seems to be a complete failure. Using fontforge 9 works better, but still has some warnings. My first ambition for this was to be able to import BW company logos. Due to ‚Äúiterate until no errors‚Äù, I'm not sure this is ready for timed dev.
I swapped the FA CSS in my build script for the new Fontello CSS; and recompiled. It is not surprising that the same font technology works in all my available test browsers (common browsers on win32, macX, Linux, Android and iPhone). In terms of user experience, this is not conspicuous; but it will load faster. This is important if you are trying to read this site whilst on the tube or underground.
Concurrently to this change, I upgraded my general font CSS to favour WOFF2, then WOFF; so all the fonts should load faster, as they are smaller files.
There is a small issue with Fontello UI, if you import a SVG glyth, you have to set the name and code value manually ~ even when these are in the SVG ~ or the symbol gets added with empty names and cannot be accessed from CSS.
Note the fontello.css doesn't set font-display 10, I would suggest adding that manually.

Conclusion

My search process hasn't showed up any alternative technology at all. This seems wrong. I have applied Fontello; as explained in the above paragraph.

There are quite a few iconsets 11, but without the smarts to convert the big panel to singular images on screen; this is limited value.
