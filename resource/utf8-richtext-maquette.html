Systems where this is useful:

  • Permanently embedding attention grabbing keywords in normal pages, be it word files, or webpages
  • Social media platforms, like twitter, where the UI doesn't include formatting.
  • Probably would work for blog type systems, that also do not support formatting.
  • Making software text-based logging easier to speed read.
  • Text messages/ SMS.
  • Text only emails, but UTF-8 (looking sideways at win32 in meaningful fashion).
  • Terminal/CLI apps (bold errors, do not use cursive on CLI its quite hard to read) .

This maquette is good as:

  • This is simple text, without html tags. It takes less bytes than a HTML equivalent.
  • It seems a rare effect at present... new features.
  • Works on recent Android devices, and recent iPhones

This maquette is poor as:

  • Look closely at the French and German samples, the accented letters are not transformed.
  • This is a one way transform; it must be done last.
  • You MUST spell check before doing the transform, as spellcheckers will not recognise words in the text effect.
  • It breaks the actual font's kerning, so its less good to read.
  • I suspect this is horrid for speech readers. UPDATE: confirmed, bad UX. But I could try aria-labelledby with a description P as a label to make it less nasty (and CSS to hide the speech reader version on visual layouts).
  • I think this will break source code tools, if used outside of comments and string literals.
  • Its use in webpages will lead to less good SEO, as its not marked up.
  • Text that may go into ingestion systems, and where the author(s) wants it to be ingested; as the text comprehension will fail (for the same pattern matching issue as the spell-check).

Other solutions

Other means to make text bold, that this maquette isn't using:

  • Changing font to a more heavy version (which should look better, especially when written at large sizes).
  • Changing CSS font-weight.
  • Using cunning layers of text at very small lateral offsets (normally done for shadows, please do not do this, it's bad accessibility for people with weak vision).
  • Making the font size larger will make it look bolder.
  • Adding a STRONG or B tag.
  • There are probably more options.
