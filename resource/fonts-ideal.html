In the 90s I was doing graphical design via Coreldraw. I was precise on typography and letter-setting. Now I can finally do the same in CSS if I prep the fonts. Con sotto voco: TODO, make each letter in the title a different font.

To get the best typography in 2024 without using kerning, or text set via a graphic system like SVG, I think apply the following:

  • Optional reading: An interesting article describing terms, not mechanisms 1
  • Have a readable font designed for your language. For non-title text check the ascenders and descenders to keep the output more readable across lines. Design-wise I like “tall text”, but this is weak accessibility. Check if it's mono-space 2 ~ which is very useful for code, less useful for plain reading-text. Unless you are doing very specific things the system fonts are quite reasonable choices. The remaining points below only apply if you are not using the system fonts.
  • For known content, tweak the font, so only the relevant glyths are included (with Fontforge 3 or similar). A web site written in Deutsch de-DE lang, doesn't need letters for ga-IE Irish or ko-KO Korean. US readers many be shocked about the number of different char-sets in their fonts.
    • If sub-setting the font file isn't an option, there is a CSS feature unicode-range to only load some glyths from a font file, if the UTF8 symbol range used are known. This is widely supported 4. This would make the font use less RAM, but have little impact on download speed. Probably most important for established Japanese and Chinese fonts which may be a 100MB large.
    • Font subsetting ranges would be better if I knew where to get a list of alphabet ranges from, 5 isn't quite the right answer.
  • Include ligature symbols in the font file for the target language (better readability, do not need to affect text for it).
    • This is extended by more CSS font-variant-ligatures, current support is quite decent 6.
  • Choose an acceptable font weight (the thickness of the strokes). There is common guidance that thin is great design, but less good for people with poor focus capacity 7. I normally make titles in a heavier weight, more urgently than bigger.
  • NOTE: Making text have a higher weight when in invert-colour mode is better for most people's vision.
  • Some people use all-caps for “attention text”. This is bad. For text of a variety of sources, consider controlling capitalisation via CSS as a final sanity check 8 9. It would be faster to fix the casing in the data storage, as it would only happen once.
  • There is no general option to adjust x-height in fonts (yet). Safari and Firefox support font-size-adjust which would be useful, but Chrome doesn't 10, ao I can't use it ~ availability 11. There is an option for harmony between different fonts 12 wide browser support but that is a different process.
  • Performance details ~ not likely to have much impact for posters, but will have for web or mobile:
    • Use the smallest font file size (woff2), so it will download faster. This reduces page loading delay.
    • Check that the font is compressed properly, there was at least one tool that doesn't use libgzip the way I would and the files are quite large.
    • Set HTTP caching headers, so the font will only be downloaded once, see basic-website-features-2021 for details.
    • Use some sort of edge-cache or CDN so there is less network latency.
    • Note, for text based websites (like a book) the text is written in advance, so there is little need for your own very fast servers, put all the computation power into the edge nodes.
  • Desired: the ability to pack several font weights and styles (e.g. italic or not) into a single font file demo. Having properly designed italic ~ rather than synthetic italic ~ reads better.
    • As the reader is less likely to be a font editor, note there are often different font files for each different weighting and amount of lean in every font purchased.
    • Example: If I load a page on a Mac, the bold text works much stronger than on my desktop. This is due to the fact that the Mac used the bold font, but my desktop faked it.
    • A simple PoC in 2012 tech 13 14, but this uses three filters, so need to change this to include ranges.
    • Better tech for mangling fonts subset
    • A tool to assist writing the CSS for loading fonts, transfonter

⚠ WARNING ⚠ this range of effort is quite involved, and will prohibit some caching services (e.g. google fonts).
