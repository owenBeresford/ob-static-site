Note the boundaries between ‚Äúcompiled‚Äù and ‚Äúnot compiled‚Äù languages are fuzzy. The most visible example being that many of my unit tests are written in Typescript, but I never have a visible compile step for them. Perl, Python and PHP do have 'object code' ~ like a compiled Clib ~ but it isn't always persisted to disk. For long-life processes it should be encouraged to-do so. For teaching purposes some people echo C code source into a bash process pipeline (echo, compiler, linker, chmod, and auto execute) ~ with a bit more sophisticated UI this would seem an interpreted language.

  • a test framework
  • a package manager
  • a lint/ prettier tool
  • best compile tools
  • docs tools
  • profilers ~ I tend to focus on DB profiling, as this is a larger slow down. But I include this section for completeness.
