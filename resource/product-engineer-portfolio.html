Mechanism maquettes

My site includes a list of tech maquettes / demos to show specific technical options. I normally create these when the technology will support “some random user” on their normal device. I propose using the tech that they present in later meetings, to avoid entrenched vendors (often reducing running costs). As a normative statement, my maquettes will age as an interaction, but they will not rot like React components do.

  • These focus on technology, not design
  • 3d-effects-maquette I wanted to explore if I could achieve 3d rotation on images in a browser. I am technical about correct rotation and projection. Answer: roughly.
  • utf8-richtext-maquette I can make letters have effects like bold without adding extra fonts, TAGS or CSS. Article includes review of use for this effect.
  • form-no-js-maquette A quite nice UI demo that uses little JS. [This item is mentioned twice]
  • transition-animation-tooltips-maquette I wanted to explore how fancy I could make a button and the trade-offs.
  • webcomponent-maquette A browser native feature, similar to React. This approach should have many fewer security issues.
  • diverse-emoji It is a historic tech limitation that smilies are all yellow. The raw UTF-8 allows a more realistic range of emoji. This maquette exists and will be kept, as support on each phone is arbitrary and incomplete.
  • colourise-emoji I explore using emoji ~ the UTF-8 chars ~ as a smaller and faster replacement for logo image files. This article looks at managing colour with the emoji, so they integrate.
  • progress-animation-maquette. I wanted to see if I could build some nice CSS animations rather than including a large GIF, MPEG or MNG file. I think progress animations should be an inline static asset, and as few bytes as possible.
  • mobile-tooltips-maquette An interaction idea, that I like.

  • I do not have a time / date picker maquette here, I did previously. This is easy npm to download.
