This is just an accumulation of notes, as certs/ SSL tech always takes me too long.

I am using:

  • ssh-keygen docs ~ A tool that easily makes standalone SSL keys. The project is focussed on supporting SSH ident easily, and uses OpenSSL code. Interesting notes, referencing other keygen tools 1.
    • Unknown provenance, but you could use 8g wifi or keytool to generate SSL keys when running win32 without admin.
    • As noted above, this is useful for ident certs on SSH login. The certs are not linked to a public rootCA.
  • XYZ library in language ABC, whilst building Apps, for the normal SSL on data-transit. See relevant package manager for more details.
    • Certs here can be overwritten/ installed on startup, or just rented as part of the service.
    • See other items for choice on rootCA.
  • openssl docs docs a cheat sheet 2 as OpenSSL is quite a large tool ~ (or more precisely, a project page of links to docs) git ~ when using SSL programmatically, but at a lower level.
    • If you have specific advanced needs this is the best option. This is OSS, and vetted source.
    • Can use private rootCA, self-signed, or public rootCA.
  • Lets Encrypt docs docs ~ An SSL tied to domain name, used for server side ident. Commercial services are available, but this is eating most of the market.
    • Supplies its own rootCA (which is already in browsers)
  • XYZ vendor, (see vendor website) if commercial people think it looks better.
    • e.g. godaddy etc.
    • These are their own rootCA
  • gpg docs docs ~ As I am moving source host away from MSFT Github, to an Ubuntu/ Canonical platform. This is a GNU project, but tightly integrated with Ubuntu.
    • Note the tooling seems more reliable when the pure CLI version is used. The GUI version emits certs that other orgs don't like.
    • uses Ubuntu's/ Canonical's rootCA.

There is some discussion on algorithms in ancient-TLS. Note fingerprints made by these different tools do not seem compatible, which is annoying, as it should be the same maths.
