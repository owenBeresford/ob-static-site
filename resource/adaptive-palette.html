I did a crawl on the internet/ npmjs, I cannot find a CSS framework that advertises adaptive colours. I thought it a good idea to replace the style sheet as its old and accumulated code; the newer code is smaller. If I add more usability features (e.g. dark-mode), the smaller style sheet will be necessary. This palette is actually the first CSS palette I have made, I normally use the premix colours in the framework, or extract values from the style-guide. However the adaption features are a hobbyists over-engineered solution; if you tell your browser that you a prefer an extreme on prefers-contrast, the palette will match it. It would be cool if there was a thing for “I'm in an over-saturated environment”, and put the brightness up; but I do not have access to that data programmatically.
If a legacy machine, probably win32, has prefers-contrast:forced 1; I'm not sure how to handle this. The flag states the user has set colours, but doesn't say how to access them?

The sparkly button is an effect I thought about. I do not think this improved the UX, so I am not using it. I can see it being a popular effect on CTA buttons for things that people buy via a desktop. This effect uses the z-index, so needs to be mixed with other features carefully. Note hover effect doesn't mean anything on touch/ haptic interfaces, and is definitely bad for people who dislike motion. If you wanted a large visual effect it would be smarter to inject an image via CSS, as that would give you a larger ability to change the visuals. I think a better effect would be to make the sparkles change size as a CSS animation, say 5pt going to 12pt, then changing colour; but that is starting to feel like power point, and definitely obstructive. Art theory suggests adjusting the colour ratios to make them brighter, the way I did with the high contract button row.
The stars used here are not selectable, will be hard to be copy inside the GUI, and will not block mouse events. I don't use Calckey social media, but the stars are an effect on that.

Please see accessible-css for more notes on colour adaption. This code is written with RGBA colours. There are other colour spaces available, but this has best support in browsers and external tooling.
