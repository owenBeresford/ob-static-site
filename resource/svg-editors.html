I built this list of tools/ libraries/ packages as in 2017 I needed to edit a SVG at runtime in the user's browser; and I thought there had to be a better solution than document.getElementById.

9 local-install SVG editor libraries which seem maintained

Software without unit-tests or enough documentation is likely to consume a lot of your time, hence I avoid them. This first list is useful if you want to make a platform that includes editing SVG images (probably a graph/ chart). The goal of this list compiled in 2017 was to be able to fix SVG assets that minority browsers wheren't rendering correctly. As 90% of the browser population was correct I didn't want to swap to a JPEG or something.

  • svg-edit git available with MIT or apache licences. It runs purely in a recent browser, and is an alternative to Inkscape if you can't use that. The first team completed the project and froze it, in about 2017, at v3. A second team has taken over the project, emphasising support for today's browsers at v7.3; the second team published via NPM as svgedit and in Netify. Version 7 uses HTML5 canvases.
  • This should be linkable: If your objective is diagrams not logos, look at GoJs docs which shows as many options as google libraries; git. As a diagram editor, it also has data-binding and inline data models. It has a support QnA sub-site 1 and mentions “undo” as a feature on the main docs page. Install from NPM as gojs; they claim it integrates with common component frameworks like React.
  • Alternatively for working-document diagrams, mermaidjs is really nice 2 docs and live demo. Install via NPM mermaid or use git has a first MIT licence and it seems to have Cypress tests. It is driven by flat ASCII like markdown, and as it renders in your browser it will have a very fast iteration rate.
  • Visualiser Raphaël ~ pls note UTF8 name ~ docs and is MIT license. This is an old JS project 3, it is compared to 4. It has an O'Reilly book. It supports a version of data binding (remember this predates Angular by several years).
  • Able to transform an SVG on a readonly page: SVG.js 5, in NPM as @svgdotjs/svg.js git MIT license. I have never used this. This is designed to be small, and not have dependencies, quoting the first link, which includes a lot of docs. The project has an eponymous twitter channel. Can be used to edit existing SVG, although it would be clumsy to do this on each page render.
  • Generator svg-draw is a declarative SVG creator, you type statements in JS and it renders it for you. It's in NPM as svg-draw git the minimal docs are 6, but it has tests. It has no deps, and is under GPL licence.
  • jquery.svg 7 git, MIT license, to clarify, code by Keith Wood (This project has many forks). This version hasn't been touched in three years. Code does not have unit tests, and there are un-merged branches, which add more recent technologies. Recommend using a fork that has maintenance.
  • Able to transform SVG on a readonly page: jquery-svg git MIT license, a plugin to “A lightweight jQuery plugin to apply css styles and js scripts to a SVG which is embedded (using the <object> tag” (quoting the last link). Read the code, its a simple class. Obviously this class can be merged with other classes to achieve more complex behaviour; this doesn't directly have document editing (reference: oab1 test code).
  • Generator Inkscape (previously named Sodipodi) 8 9 sodipodi project, inkscape project is run on BZR, and I can't find the repo. This is a desktop application, mostly written for Linux, although it runs on other OS. It is OSS (GPL), and may be deployed without a charge. I put this item here, as this is the best solution for quick edits to existing files. As a Linux user, install via apt, dnf etc. Tools for other OS may vary e.g. MacOSX 10. I do not know how to link or script this option.
  • As a negative mention snapsvg 11 12, under Apache2 license. It seems to be owned by Adobe Inc (see github account) git, and they demand paper address of any contributors. As an older project, there are many open tickets. UPDATE 2023: it still has pull requests and open tickets on NPM; don't use this library as it's not maintained.

6 Online SVG editors (maybe pay-for service)

If you do not have a desktop install for image generation; but need to manipulate an image, then these are quite useful. This list is focussing on SVG format again. In theory you could run these on your phone; but I suspect you would get better experience with a bigger screen. I checked a few details for each editor, if you are a full-time designer/ illustrator/ animator you are likely to have a long detailed list of features that you need. NB: There are many many articles listing SVG editors online, I added this extra section as it showed as a frequent Google search term. As other notes say you can only scale SVG to a few thousand polygons, drawing via a webrowser doesn't make this limit lower; sentence being biased as 3d rendering engines use many more polygons, but they are all triangles.

  • Miro 13 14 ✓ export to SVG ✓ import SVG ✓ basic transforms ✓ precise dimensions ✓ supports free accounts 15. This supports good enough free hand drawing with a mouse to be able to sketch workmates with (assuming you can draw).
  • Vecta 16 17 ✓ export to SVG ✓ import SVG ✓ basic transforms ✓ precise dimensions 18 ✓ supports free accounts 19
  • Canva 20 21 ✓ export to SVG ✓ import SVG ✓ basic transforms ✓ precise dimensions 22 23 ✓ supports free accounts for education, look at these for feature breakdown 24 25
  • SVG editor ✓ export to SVG ✓ import SVG ✓ basic transforms X no precise dimensions ✓ supports free use, with no account. I cannot find docs for this due to the incredibly SEO name.
  • Draw 26 27 28 ✓ export to SVG ✓ import SVG ✓ basic transforms ✓ precise dimensions ✓ supports free accounts 29 30. This has the look and feel of a desktop app from the 90s, not a webpage/app. It loads quite slowly ~ about 5s, but after that is very smooth and responsive. This is a SVG diagram editor, and does support freehand lines 31.
