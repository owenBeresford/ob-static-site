Language target

SQL has several specs 1 2, and even ISO rating 3. The current newest spec is from '23, and has been sub-divided 4. An out of date '92 SQL spec 5 ~ but is open access. Every DB vendor will be running their own variant.
SQL and related languages are server-side data languages, and I think fairly functional as declarative. SQL performance is a function of the data it is applied to, and the schema ~ databases always gain with more RAM.

This section here is normally where to get versions of the language from. This is harder for DB as there are many vendors. Some auto-indexes if you actually need to know 6 7 ~ RHEL has many fewer in core repos. Any Win32 offering for package management as a normal feature rather than an special enterprise addon? naah. As a win32 user, you will need to use the internet at random, and hope you don't install a crypto miner. There are many cloud vendors offering DBaaS capacity [...].

I am not aware of package managers for database platforms. UPDATE: Suprabase runs a catalogue for a PG extensions git docs. If you install the DB client package to your local machine, there is normally a procedure for including and sometimes running scripts on client startup. As an idea, this is just a resource file for the client process, like a `.bashrc`. Users can create complex STORPROC type logic in tools that do not support STORPROC.

The SQL language will be interpreted by the client (for validation) AND the server. When the DB is accessed from a software the client-side validation will be narrower.

Tools

  • An IDE: normally programming IDE normally support SQL without any extra effort by you. DBeaver community edition is an opensource option git docs.     • Even in locked-down spaces, SQLite3 can probably be placed and used (as no server process).
    • Obviously many DB can be installed and their clients can be installed to an average work-laptop if the data volumes are moderate.

    • There is TAP support for PG pgTAP. Separately, according to 13, 14 PG supports ASSERT and setting terminal colours as SQL.

    • T-SQL ~ a local service ~ obviously focussed on MSFT T-SQL docs 17, git, licence: Apache 2.0 . A walk-through 18.

    • Permissions, for payment systems I secure critical fields in addition to tables if possible
