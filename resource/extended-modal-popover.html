Please also see modal-popover and zoomable-widget for more work. This maquette is a fairly trivial extension so the widget looks more like some other websites I have seen.
The change is just CSS. There are no images for logos, and the JS is just for UX niceties that CSS can't touch.

Its not the standard widget edition (on my site), as this site doesn't use many curves. I need to add a few more details before the maquette is finished. The new section of CSS was about an hour, maybe two. I didn't see any good HTML solutions in NPM. It is practical to use React to make HTML islands, but handover most of the runtime work to the CSS engine (which is C/ C++ code, and much faster).
I am not aware of CSS classes in standard frameworks that would deliver this effect, but as I was building a singular specific thing, I admit I didn't look that much. A sensible place to look would be a CSS animation framework, but depending on other animation in the app-under-dev that may be overkill.
Very important note: this minor interaction DOES NOT block the UI in any fashion, and its also not selectable, if you grab all the text. JS animation often does.
