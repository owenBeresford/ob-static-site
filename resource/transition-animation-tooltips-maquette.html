This is not a 'shiny new tech' demo, it's just a config alternative. My rich tooltips library supports two rendering styles, a) CSS display option, b) CSS visibility option. The 'display' option is faster to render, and so better accessibility, particularly on large webpages. The 'visibility' option allows animation, in particular transitions. I tried the visibility option first, but it was obstructing page rendering, so I needed to swap.
Note: this will work in Firefox as soon as you can use FF b126 or greater.

I ought to extend this to work with entry-exit-animations CSS options 1. The range of support is a bit low for production 2 at 88%.

Edge cases.

  • Transition animations do not work for percentage sized measures 3
  • Transitions animations are brittle with values involving CSS variables 4 (and another maquette)
  • Transition animations are not run in Chrome for HTML in a subdocument, like the ShadowDOM. This is avoided by ensuring a relevant component has focus when you want to run the animation.
