It is a common feature in e-commerce to have an image display of your sales goods; but also have a widget to show a zoomed in image.
Article not focused on mobile but technology should work in either hardware platform. A close up version of a sales-good's image is probably needed for mobile UI, if it is a tangible item; as a well proportioned image is likely to make it harder to see the details.

Interactions to access zoomed data

The zoom technology could be accessed by one of the following access methods (i.e. interactions I propose are supported):

  • Hover cursor over image (best for large screen/ WIMP device)
  • Tap image (best for small screen haptic device)
  • Swipe off other images, and have auto-packer increase size the remaining one(s)
  • Scroll when cursor is on image
  • Stretch/ pinch image
  • Select a value in a select-box (i.e. like zoom in office applications)

How to zoom in a webpage/ webapp?

I workout several methods to have a zoom and clip of an image.

  • Simple CSS for image static scaling: object-fit 1, object-position 2. This is really useful for browsers where it works 3, and should be used for all RWD / PWA designs 4. But this is not a popup/ is not reactive to user input.
  • CSS transform:scale 5 6 older browsers have this feature as zoom 7 8, but that is deprecated, just like MSIE that started that CSS item. This should probably be combined with overflow-x/ overflow-y 9 to avoid too many scrollbars.
  • If using a desktop/ laptop, <cntl + mouse-scroll> will normally zoom. With many mobile apps pinch will zoom. Both these are operating system features, but I use them a lot.
  • “blitting” or Sprite mapping is an old pre-internet technique for pulling a currently wanted texture from a big texture file; and applying it to what is currently on screen 10. I have been shying away from doing this is JS/CSS, as I didn't see any libraries, and I didn't want to hand calculate all the maths, 11 12 has some ideas to avoid manual typing. Support is less sharp for this feature 13 (looking at colour of support rather than volume).
  • A low tech/ inefficient use of bandwidth width, but very stable and very reliable trick is to shoot all you images in very high resolution; and just set the “normal size” image to have fixed pixel sizes; the browser will scale the image 14. Then the popup can use the same image at real size.

As an a11y feature, it would be nice to read any browser contrast requests, and make photos more intense if users have logged this preference. CSS contrast and brightness 15 16 can be applied to images, and has good support. As an invisible benefit, adjusting contrast on the server often makes JPEGs smaller files 17, not relevant if you are using WEBP or something.

Cheat solutions

  • Use a PICTURE element, and an array of images 18 19. Set a display criteria; and the browser will just render the best one.
  • Images of sales goods are normally photos, so some type of JPEG. If your data started as a SVG (say you need to display a image of the London tube map); you have a wide level of control 20. SVG is a scalar image format, and includes the co-ordinate space limits in its outermost tag. These can be set, via JS, to give a zoom. Remember when zooming SVG that line thicknesses do not magically scale with the image, and will need to be incremented. When pulling data for the article, I found Svidget 21 git, docs site has expired cert, so I'm not linking to it. See also 22

A Sketch of a useful component:

A widget that accepts pinch/ stretch interactions, scroll interactions, an image URL, an optional style object and a range of params/ param struct. The listed interactions change the zoom level on the image; if a level uses more more pixels that the builtin Container can accommodate, the image will be clipped.
The amount of zoom could be set by a CSS variable; unfortunately, set change the CSS var at runtime; JS is still needed 23 24
There are a variety of things in NPM that are similar, but nothing that is what I want NPM search results
