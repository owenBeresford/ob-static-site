Language target

Elixir doesn't seem to have a formal spec, but the master project implementation is the equivalent reference point. There is a syntax reference 2, and list of published [to the interpreter] kernel features 3 and the kernel 4 module. A comparison of Elixir implementations 5. Related information, some notes for Erlang 6 7. Due to conceptual model, types are a bit different 8 9 10, but these are in the process of being changed 11.
Elixir is a server-side functional focussed general-purpose programming-language 12, and is used for ‚ÄúHygienic metaprogramming by direct access to the AST‚Äù. The bytecode for Elixir is seamlessly compatible with Erlang's interpreter, the BEAM engine 13 which Elixir uses as a runtime interpreter. Due to early use of Erlang in telecoms, there is OTP 14 15 ~ Open Telecom Platform ~ this is a middleware and supplies extremely-concurrent-processes tools.

The new version of the basic language from 2024/12 is 1.18 16 which adds more type support. The newest version in Ubuntu 24 is Elixir 1.14, and Elixir website 17 recommends running `add-apt-repository ppa:rabbitmq/rabbitmq-erlang` to get a newer edition. Windows install process 18, the other operating systems are on the same page.
If running complex Elixir infra, look at 19, for version managers, same structure to `nvm`. The highest recommended one is asdf. The end-of-life plot for Elixir 20. The equivalent to `packages.json` in JS is `mix.exs` 21

In some languages the compiler is a licensing problem, and the market will present several. Elixir has `elixirc` 22. Compilation can be done from the basic Elixir shell, here is a tidied0up UI esbuild docs. A compiler to native code 23 in the package manager.
